# 迟滞带 lag 校准 A/B 扫描（C 档科技阻尼器）

**窗口**：[['2022-01-01', '2022-12-31'], ['2024-01-01', '2026-07-17']]；**成本**：8bps；**lookback**：20日

**目的**：为「进档快/出档慢」迟滞带选最小可行 lag——既保留换手收益（迟滞带仍平滑预算抖动），又不损反弹（相对无带 raw 的收益缺口最小）。raw(lag=1)=无带参照上限。

## 判定

- **推荐 lag = 2**
- lag=2 仍节省 1.7 换手（迟滞带仍平滑预算抖动），且反弹收益缺口最小（SOXX 8.21% / QQQ 2.8% vs raw）；回撤保护与其他 lag 完全相同（-13.43% / -7.7%，入档机制一致）。为最优解：在保留换手收益的同时最不损反弹。

## 方法论与局限

- 复用 backtest_long_run_dampener 的加载器与 live kernel；迟滞带仅改 `_attach_soxx` 的 lag。

- 2022 HY 为月度锚代理、DXY 两段时间量纲不同（已在前述文档明示），baseline-vs-阻尼 对比内部一致。

- 代理口径：预算=毛敞口、降仓收益记0、成本=|Δbudget|×8bps，未含滑点/融券/税费。
